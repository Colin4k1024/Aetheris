## 1. 项目概况

### 1.1 项目定位
- **项目名称**: Aetheris (CoRag)
- **定位**: 智能 Agent 执行运行时 — "AI Agent 的 Kubernetes"
- **核心价值**: 为 AI Agent 提供持久化、可回放、可观测的执行环境
- **类比**: Temporal for Agents — 专门为 AI Agent 设计的分布式任务运行时

### 1.2 技术栈

| 组件 | 技术选型 | 版本 |
|------|----------|------|
| 语言 | Go | 1.26.1 |
| Web框架 | Hertz (Cloudwego) | v0.10.4 |
| Agent框架 | Eino (Cloudwego) | v0.7.29 |
| 数据库 | PostgreSQL | - |
| 缓存 | Redis | go-redis/v9 |
| 链路追踪 | OpenTelemetry | v1.40.0 |
| 配置管理 | Viper | v1.19.0 |
| 协议 | gRPC + HTTP | - |

### 1.3 代码规模

| 指标 | 数值 |
|------|------|
| Go源文件 | ~400+ |
| 内部包 (internal/) | 20+ |
| 公共包 (pkg/) | 15+ |
| 测试文件 | 覆盖主要模块 |
| go.mod 依赖 | 22个直接依赖 |
| go.sum 依赖 | 264个传递依赖 |
| 代码总行数 | ~10万行+ |

### 1.4 核心功能

✅ **事件溯源**: Job执行全程记录，支持回放和审计  
✅ **At-Most-Once执行**: 工具调用不重复，即使崩溃后恢复  
✅ **检查点恢复**: 从检查点恢复执行，不从头开始  
✅ **人类审核**: 支持 Human-in-the-Loop 暂停和审批  
✅ **多框架支持**: 可接入 LangChainGo、LangGraphGo、Eino ADK  
✅ **企业级**: RBAC、合规性报告、数据主权、PII脱敏

| 人工审批 | Human-in-the-Loop 暂停机制 |

| 竞品 | Aetheris优势 | Aetheris劣势 |
