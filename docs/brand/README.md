# Aetheris Brand Guidelines

This document defines the brand identity, visual guidelines, and messaging standards for Aetheris.

## Brand Overview

**Aetheris** is a durable, replayable execution runtime for production AI agents — the "Temporal for Agents" that your production AI systems desperately need.

### Tagline
- **Primary:** "The Missing Layer for Production-Ready AI Agents"
- **Secondary:** "Kubernetes for AI Agents"

### Mission Statement
To make AI agents production-ready by providing durability, reliability, and observability for enterprise deployments.

## Visual Identity

### Logo Usage

The Aetheris logo represents durability and layering (like layers of atmosphere).

```
[Aetheris Logo Placeholder]
```

**Do:**
- Use the official logo on dark or light backgrounds
- Maintain clear space around the logo (minimum 1x height)
- Use the correct color variant for the context

**Don't:**
- Stretch or distort the logo
- Add effects (shadows, gradients) not in the original
- Place on cluttered backgrounds

### Color Palette

| Color Name | Hex Code | Usage |
|------------|----------|-------|

- **Headings:** Inter (Bold)
- **Body:** Inter (Regular)

## Messaging Guidelines

### Tone of Voice

Aetheris communications should be:
- **Technical but accessible** — Clear explanation without oversimplification
- **Confident but not arrogant** — Proud of capabilities without disparaging competitors
- **Professional yet human** — Industry-appropriate but not corporate-speak

### Key Messages

#### For Developers
> "Your AI agent worked in testing. But production is different. Aetheris runs them in production."

#### For Enterprise
> "Event-sourced traceability, replayable evidence, and compliance-ready audit trails for AI agents."

#### For Operations
> "At-most-once tool execution, crash recovery, and deterministic replay — the reliability your AI systems need."

## Content Guidelines

### Naming Conventions

- **Project name:** Aetheris (always capitalized)
- **Components:** Use camelCase for internal components
- **Features:** Capitalize feature names (Checkpoint, Ledger, Effect Store)

### Writing Style

1. Use active voice
2. Be specific with technical details
3. Include code examples where relevant
4. Keep sentences short and clear

### Terminology

| Term | Usage |
|------|-------|
| Agent | An AI-powered autonomous entity |
| Job | A durable task unit |
| Checkpoint | State snapshot for resume |
| Effect | External side effect |
| Ledger | Authorization record for at-most-once |
