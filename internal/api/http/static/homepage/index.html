Why teams adopt Aetheris

Production agents rarely fail in the happy path. They fail mid-run, retry side effects, and leave teams without a clean audit trail. Aetheris wraps those failure modes in durable execution semantics so you can resume work, prove what happened, and keep external actions idempotent.

01

Crash recovery

When a worker dies, the next worker resumes from the last durable checkpoint instead of restarting the entire task.

02

At-most-once execution

Payments, emails, and order creation stay protected by an invocation ledger so retries do not duplicate side effects.

03

Replayable audit trail

Every job transition can be traced and replayed for debugging, compliance, or post-incident analysis.

Quickstart

Launch the API locally, then confirm the runtime is reachable before wiring in your own agent.

git clone https://github.com/Colin4k1024/Aetheris.git
cd Aetheris
make run-embedded

curl http://localhost:8080/api/health
What you can do next
  • Submit work to POST /api/agents/:id/message
  • Inspect jobs through /api/jobs/:id/trace/page
  • Use the existing quickstart and adapter docs from the repository
