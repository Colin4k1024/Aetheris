## Content Templates

### Technical Comparison Post

```
TITLE: [Topic]: [CoRag Approach] vs [LangChain/LangGraph Approach]

STRUCTURE:
1. Problem statement (1-2 paragraphs)
2. How CoRag addresses it (with code)
3. How LangChain addresses it (with code)
4. Comparison and tradeoffs
5. When to choose each
6. Key takeaways

LENGTH: 1500-2500 words
TONE: Educational, not competitive
```

### Integration Guide Template

```
TITLE: Using [CoRag Feature] with LangChainGo/LangGraph

STRUCTURE:
1. Overview of the integration (why useful)
2. Prerequisites
3. Step-by-step setup
4. Example code
5. Common issues and troubleshooting
6. Resources for more

LENGTH: 800-1500 words + code samples
```

## Cross-Promotion Guidelines

### When to Cross-Post

✅ **Appropriate:**
- Technical tutorials that benefit both communities
- Integration guides for using both tools
- Announcements of joint initiatives
- Community highlights featuring members from both

❌ **Not Appropriate:**
- Pure CoRag feature announcements in LangChain channels
- Promotional content without value to the other community
- Duplicate posts without adaptation

### How to Cross-Post

1. **Adapt to audience**: Don't copy-paste; tailor the message
2. **Add context**: Explain why this is relevant to that community
3. **Invite engagement**: Ask questions, don't just broadcast
4. **Credit sources**: When sharing other's content, give credit

### Language Guidelines

| Channel | Language | Notes |
|---------|----------|-------|
| LangChain Discourse (English) | English | Formal-ish |
| LangChain Discord (English) | English | Casual |
| LangChainGo Discord (Chinese) | Chinese | Casual, familiar |
| CoRag channels | English or Chinese | Match existing |

```
Idea → Draft → Internal Review → Partner Review → Publish → Promote
  │        │            │              │           │         │
  ▼        ▼            ▼              ▼           ▼         ▼
Submit   Write      CoRag team      LangChain   Post to   Share in
form     first      comments        team        blog/      both
         draft                                    forum     communities
```

### Review Process

1. **Internal draft**: Written by CoRag contributor
2. **Internal review**:另一位 CoRag team member reviews
3. **Partner review**: Sent to LangChain contact (5 business days)
4. **Revisions**: Incorporated based on feedback
5. **Publish**: Coordinated timing announced
6. **Monitor**: Comments and feedback tracked
