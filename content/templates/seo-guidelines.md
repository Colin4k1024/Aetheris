# SEO 优化指南

针对中文技术平台（掘金、知乎）的 SEO 优化指南。

## 平台 SEO 特点

### 掘金 (juejin.cn)

**优势**：
- 技术文章权重高
- 标签系统完善
- 推荐算法友好

**优化重点**：
- 标题关键词前置
- 标签选择精准
- 配图质量
- 完读率

### 知乎 (zhihu.com)

**优势**：
- 百度权重高
- 长尾词排名好
- 问答形式易收录

**优化重点**：
- 标题问题化
- 内容深度
- 外部引用
- 互动率

## 标题优化

### 掘金标题公式

```
[类型] 核心关键词 + 副标题 + 平台标识

示例：
- 【入门】5分钟上手 Aetheris：AI Agent 持久化运行指南
- 【实战】Aetheris + LangChain：构建生产级 Agent 系统
- 【对比】Aetheris vs Temporal：选型指南（2024）
```

### 知乎标题公式

```
问题型：如何/怎么 + 关键词 + 问题
分析型：为什么 + 关键词 + 分析
教程型：关键词 + 从入门到精通

示例：
- 如何用 Aetheris 解决 AI Agent 的崩溃恢复问题？
- 为什么 AI Agent 需要专门的持久化运行时？
- Aetheris 入门指南：从零构建可靠的 Agent 系统
```

### 标题检查清单

- [ ] 标题长度：掘金 30-50 字，知乎 20-40 字
- [ ] 核心关键词在标题前 10 字内
- [ ] 避免：标题党、夸张词汇、纯品牌词
- [ ] 包含：数字（如"5分钟"）、动作词（如"构建"）

## 内容结构优化

### 段落结构

```
# H1 标题（包含关键词）

## 简短前言（100字以内）

## H2 第一部分
内容...

### H3 小节（可选）
内容...

## H2 第二部分
...

## 总结
- 要点回顾
- CTA
```

### 段落长度

| 平台 | 每段建议 | 每篇建议 |
|------|----------|----------|
| 掘金 | 2-4句 | 1500-3000字 |
| 知乎 | 2-5句 | 2000-4000字 |

### 列表和表格

- 使用有序列表：步骤、流程
- 使用无序列表：特点、要点
- 使用表格：对比、规格

## 代码块优化

### 掘金代码块

````markdown
```go
func main() {
    fmt.Println("Hello Aetheris")
}
```
````

### 最佳实践

1. **语言标注**：始终标注语言类型
2. **标题**：重要代码块添加标题
3. **高亮**：关键行使用高亮

````markdown
```go {3}
// 关键代码行高亮
func example() {
    result := process() // highlight
}
```
````

4. **行号**：长代码添加行号

## 配图优化

### 图片要求

| 平台 | 建议尺寸 | 最大大小 |
|------|----------|----------|
| 掘金 | 1280x720 | 5MB |
| 知乎 | 900x600 | 5MB |

### 图片命名

```
good: aetheris-architecture-diagram.png
bad: image001.png, screenshot_2024.png
```

### Alt 文本

所有图片添加描述性 alt 文本：

```markdown
![Aetheris 系统架构图](./images/architecture.png "Aetheris 系统架构图")
```

## 外部 SEO

### 被引用

1. **在技术博客发布原文**
2. **提交到导航站**：如程序员盒子、前端导航
3. **社交媒体分享**：Twitter、微博、V2EX

### 质量信号

| 信号 | 说明 | 提升方法 |
|------|------|----------|
| 点击率 | 搜索结果中的点击率 | 优化标题和描述 |
| 停留时间 | 读者在页面停留 | 提高内容质量 |
| 互动 | 评论、点赞 | 引导互动 |
| 分享 | 被转发分享 | 优质内容 |

## 技术 SEO

### 页面加载

- 图片压缩（< 200KB）
- 代码高亮使用 CDN
- 避免大量内嵌脚本

### 移动端

- 响应式布局
- 点击区域足够大
- 字体大小合适（≥14px）
